LockSentry - Master User Guide

Overview

LockSentry is a modular PowerShell-based automation system for managing SMB file handles and sessions. This guide provides a comprehensive explanation of how to configure, execute, and extend each component of the system.

  • Manage-Modules.ps1 â€“ Installs, removes, or lists PowerShell modules based on Modules.json.
  • Clean-Logs.ps1 â€“ Archives and purges logs based on age and configuration settings.
  • Manage-SmbFiles.ps1 â€“ Executes SMB file handle scanning and cleanup via SmbFileManager.
  • Manage-SmbSessions.ps1 â€“ Executes SMB session filtering and closure via SmbSessionManager.

Configuration Files

Each environment (dev or prod) contains the following files:

  • Environment.json â€“ Specifies the active environment via CurrentEnv.
  • RunPlan.json â€“ Array of steps to run, where each step includes a script name and a repeat count (Times).
  • Modules.json â€“ Module list with source paths and names for installation.
  • LogCleanup.json â€“ Controls archiving and deletion logic (e.g., ArchiveAfter, DeleteAfter).
  • SmbFiles.json â€“ Controls file handle scan settings: directories to scan, log behavior, mode, and user filters.
  • SmbSessions.json â€“ Controls session scan behavior including mode, logging, and user inclusion/exclusion.

Workflow and Execution Order

LockSentry-RunPlan.ps1 acts as the main task runner. It:

  1. Determines environment using Environment.json
  2. Performs log cleanup using Clean-Logs.ps1 and LogCleanup.json
  3. Installs modules using Manage-Modules.ps1 and Modules.json
  4. Loads and executes each step in RunPlan.json using the appropriate config:
    • Manage-SmbFiles.ps1 â†’ SmbFiles.json
    • Manage-SmbSessions.ps1 â†’ SmbSessions.json

Each step may be repeated using the Times value in RunPlan.json.
